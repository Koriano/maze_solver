Bienvenue dans notre projet sur la résolution d'un labyrinthe. 

Dans ce dossier vous trouverez différents éléments:

	- Le dossier images, qui est un répertoire contenant diverses images que nous avons 
	utilisées. Vous n'aurez pas besoin de l'utiliser.

	- Le dossier libs : ce répertoire contient les différentes librairies qui nous seront 
	utiles pour la résolution de nos différentes parties. 
		
	- ComparaisonPartie1-2 : ce script va permettre de comparer les parties 1 et 2
	de notre projet. Si vous souhaitez le tester, sachez que son execution met un 
	certain temps car il génère des matrices de grandes tailles plusieurs fois. 

	- Part1 : ce script concerne la première partie du projet. Afin de le tester, 
	il vous suffit de l'executer, de saisir la hauteur et la largeur de matrice que
	vous souhaitez. Le script vous informera de l'avancée de la résolution ainsi
	que du temps d'execution de cette dernière. 

	- Part2 : ce nouveau script concerne la deuxième partie de notre projet. Tout comme 
	le précédent, lors de son execution il vous suffit de rentrer la largeur et la 
	hauteur de labyrinthe que vous voulez. Le script vous fournira les mêmes informations 
	que le précédent. 

	- Part3 : ce dernier script cloture notre projet. Son execution est un peu plus délicate
	que pour les parties précédentes. 
	Voici donc la démarche à suivre :
		
			Executer le script
			Une première fenêtre va s'ouvrir : sur cette dernière il vous faudra
			rentrer la taille de labyrinthe que vous souhaitez (matrice carrée).
			Il vous faudra également choisir le nombre de génération maximum que 
			vous acceptez, c'est-à-dire à partir de combien de générations vous 
			considérez que le labyrinthe prend trop de temps pour se résoudre. 

			Il est conseillé de choisir 500 générations lorsque vous ne choisissez
			pas d'appliquer l'optimisation et 5000 lorsque vous souhaitez l'appliquer
			(ces valeurs sont uniquement indicatives)

			Pour terminer si vous souhaitez résoudre le labyrinthe avec l'optimisation
			décrite dans le rapport il vous suffit de cocher la case. 


			
			Après avoir saisi ces données vous allez devoir valider. Puis une deuxième 
			fenêtre s'ouvrira et vous indiquera que la résolution est en cours. 
			Une fois que la résolution sera terminée un bouton suivant apparaitra. 
			Il vous faudra appuyer sur ce bouton pour accéder aux résultats. 

			Si le labyrinthe n'a pas de solution (trop long ou nombre de génération maximum
			trop faible) une fenêtre s'ouvrira en vous signalant qu'il n'y a pas de solution.
			Sinon, le chemin trouvé ainsi que la fonction fitness s'afficheront à votre écran.

			De plus, la console affichera en temps réel le nombre de générations et le temps 
			d'execution de chacune. A la fin de l'execution, le temps total ainsi que le nombre 
			de plateau détectés s'afficheront également dans la console. 

 


	Si vous ne souhaitez pas utiliser l'interface, ou que vous rencontrez des difficultés pour son execution,
	un code est mis en commentaire dans la partie main du programme. Il vous suffit de le décommenter pour 
	le visualiser. 

	Toutes les images affichées seront enregistrées dans votre répertoire courant. 



Vous disposez maintenant de toutes les informations nécessaires afin d'executer correctement nos différents codes
et de les tester à votre guise.
